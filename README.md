# Kodi 控制器切换插件 (Controller Switcher)

这个插件旨在帮助用户快速自由的在kodi内切换不同的遥控器或其他控制设备。

## ✨ 核心特性

### 1. 广泛适配的遥控器列表
适配了市面上多款主流电视盒子和外设遥控器，实现即插即用（不同平台部分遥控器可能有差异，主要针对 CoreELEC/Linux 环境）：
- **芝杜V12** (Zidoo V12)
- **华为R22** (Huawei R22)
- **移动高清语音遥控**
- **G20sPRO**
- **Homatics 标准遥控器** (无数字和四色键)
- **Homatics 高级遥控器** (带数字键和四色键)
- **UGOOS UR02** / **UGOOS UR01** (AM6B Plus 原装)
- **小湃 RC173S** (腾讯极光商城出售的遥控) / **RC173** (腾讯极光6S原装)
- **标准键盘** / **通用最小适配** 


### 2. 强大的自定义按键映射 (Custom Keymapping)
- **全自由度的配置方式**：插件在加载遥控器的默认配置文件时，会自动加载其附带的底层硬件映射文件（`.hwdb`，如有）以及默认按键映射文件（可选）。在自定义选单中，用户既可以选择提取这份默认配置并在此基础上大改特改，由于底层按键已被 `hwdb` 规范化为 Kodi 能理解的标准环境，使得修改十分顺手；当然，用户也可以选择纯手动起步，直接在空白基础上仅对特定按键进行精准的自定义重映射。
- **利用 Kodi 原生机制合并配置**: 针对当前遥控器，插件采用了首字母排序命名法隔离配置文件：遥控器的默认按键方案以 `a-` 开头命名（例如 `a-<remote>.xml`），用户的自定义按键文件以 `z-` 开头命名（例如 `z-<remote>-overwrite.xml`）。Kodi 引擎在读取 `keymaps` 目录时，会遵循按文件名**字母顺序依次加载**的规则，后加载的文件节点会自动覆盖先加载文件中的同名节点。通过这种机制，原生的 Kodi 就能完美地将你的自定义键位（`z-`）覆盖到默认配置（`a-`）上，既保留了基础方案的完整性，又实现了自定义修改的灵活独立。

### 3. 完整适配我的其他插件的接口
- 默认映射包含了跳过片尾,一键字幕音轨的映射,同时自定义映射中也列出了这些接口以便自定义绑定

## 🚀 使用方法

1. **选择遥控器**：在 Kodi 插件界面中打开本插件，从列表中选择你正在使用的控制器/遥控器。
2. **加载默认配置**：选中遥控器后，系统会自动部署它所需的硬件配置文件（如 `.hwdb`），并加载针对它专门优化好的默认kodi按键映射文件（可选）。
3. **查看适配说明**：强烈推荐您在配置完毕后，查看对应遥控器的“适配说明”。它会告诉你各个按键对应作者精心设计的操作逻辑，让你快速上手。
4. **自定义按键覆盖**：如果在实际使用中，有某些按键动作你并不满意或不符合个人习惯，请进入“编辑自定义适配文件”页面，专门为这个按键编辑自定义绑定。系统将自动生成自定义覆盖文件（`z-<remote>-overwrite.xml`）并无缝覆盖你不满意的默认绑定，而其他的默认按键不受任何影响。
